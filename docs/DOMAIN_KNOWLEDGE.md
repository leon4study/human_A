## 1. 스마트팜과 양액 시스템

### 1-1. 스마트팜의 정의
ICT 기술을 통해 생산성을 극대화하고 경영 비용을 절감하는 **'지능형 농업 생산 공정'**. 단순히 기술을 접목한 농장이 아닌, 데이터 기반 운영 시스템.

**4대 핵심 구성요소**: 센서장치 / 제어장치 / ICT 플랫폼 기술 / 생산 설비

### 1-2. 양액 시스템 = 스마트팜의 심장
사람의 혈액을 뿜어내는 심장처럼, **양액 시스템은 재배지 모든 작물에 골고루 양액을 제공**해 균등한 고품질 작물 수확을 가능하게 합니다.

작동 원리:
1. **원수 탱크**의 깨끗한 물 + 작물 성장에 필요한 **다양한 영양분** (A액·B액)을 미리 설정해둔 최적 비율로 혼합
2. 잘 만들어진 **배양액**을 관수관(파이프)을 통해 재배지 곳곳에 균일하게 공급
3. **점적관수(drip irrigation)** 방식으로 물방울 단위 정밀 공급

### 1-3. 점적관수 + 압력보상형 노드팁 (CNL 핀)
국내 스마트팜 **90% 이상이 점적관수 방식** 사용. 대부분 **고형배지경(코코피트·암면·펄라이트) + 점적관수** 조합.

```
양액펌프 → 메인 호스 → 일정 구간마다 압력보상형 노드팁(CNL 핀) → 물방울
            ↑
       내부 압력이 Cracking Pressure(예: 1.0 bar) 이상 도달 시
       핀이 동시 개방 → 균일한 물방울 배출
```

**CNL (Compensated Non-Leakage) 핀의 핵심 특성**:
- 내부 수압이 **임계 압력(Cracking Pressure ≈ 1.0 bar) 이하**일 때는 한 방울도 안 내보냄
- 임계 압력 돌파 시 동시 개방 → 압력 변동에도 균일한 유량 보장
- 핀 종류는 다양하지만 모두 동일 원리

**장점**: 시간·양 정밀 제어 가능, 물 절약, 균일 공급
**단점**: 핀 막힘에 취약 → 본 프로젝트가 해결하려는 문제

## 2. "막힘"의 메커니즘

### 2-1. 점적 핀이 왜 막히는가
관수 중단 시간 동안 **양액 속 수분만 증발하고 화학 성분(염분·석회)이 남아 결정화**되어 핀 내벽에 침전:

| 원인 | 메커니즘 |
|---|---|
| 화학적 (염분·석회·미네랄) | 칼슘·마그네슘 등이 **건조 시기**에 침전물 형성. 관수↔중단 반복으로 내벽에 점착력 증가하며 층층이 누적 |
| 생물학적 (이끼·바이오필름·미생물) | pH·온도 상승 시 미생물 군집체가 관내 형성, 유로 차단 |
| 물리적 (토사·이물질) | 원수 자체에 포함된 입자 |
| 운영적 (A/B액 비율 오류·세척 미실시) | 사람 실수로 인한 농도 이상 → 결정화 가속 |

### 2-2. 막힘이 데이터에 발현되는 흐름
```
점적 핀/관로 막힘 (물리적 원인)
   ↓
관로 저항 ↑ → 차압(differential pressure) ↑
   ↓
유량(flow_rate) ↓
   ↓
펌프가 목표 유량 유지하려 더 큰 부하
   ↓
모터 전류·전력·온도 ↑ → 베어링 마모 가속
   ↓
정지 시 임펠러 손상 → 펌프 수명 급감
   ↓
재배사 전체 작물 동시 수분 미공급 → 회복 불가능한 스트레스
```

→ **유량 감소가 가장 빠른 1차 신호**, 모터 부하 변화는 후행 지표.

### 2-3. 핀 막힘의 "심각한 위험" 수질 기준 (논문 기반)
[MDPI 논문](https://www.mdpi.com/2071-1050/17/9/3981) 기준 — 다음 수치 초과 시 핀 막힘 치명적:

| 항목 | 임계치 |
|---|---|
| 부유 물질 (Suspended solids) | > 100 mg/L |
| 전기전도도 (EC) | > 3.0 dS/m |
| 용존 고형물 | > 2000 mg/L |
| 질소 (NO3-N) | > 3 mg/L |
| 총 철분 | > 1.5 mg/L |
| 망간 | > 1 mg/L |
| pH | > 8.0 |
| 나트륨흡착비 (RAS/SAR) | > 9 |

### 2-4. 막힘 방지 운영 전략
- **관수관이 건조해지는 비가동 시간 최소화** (침전은 정지 시 발생)
- 다이어프램 펌프는 저복사량(~200 W/m²)에서도 가동 가능 → 흐린 날·아침/저녁에도 운영
- 정기 플러싱·산처리·염소처리·여과 강화

## 3. ⭐ 밤→낮 기동 시 압력 메커니즘 (중요)

본 프로젝트의 **"startup spike를 정상으로 봐야 하는 물리적 근거"**.

### 3-1. 밤 동안 일어나는 일
1. 야간엔 관수 중단 (작물 휴식)
2. 배관 끝단(핀 옆)에 맺혀있던 양액에서 **수분만 증발**
3. 자리에 **하얀 비료 소금기(염류)가 딱딱하게 결정화** → 미세한 핀 구멍 막아버림

### 3-2. 아침 펌프 기동 시 발생하는 그래프
```
시간     | 물리 현상                           | 압력 그래프
---------|-------------------------------------|------------------
0~1분    | 펌프 ON, 물 들어옴 + CNL 핀 닫힘    | 수직 상승 💥 (Spike)
                                                    │
                                                    │  ← 미세 소금벽까지 부숴야 함
                                                    │
1분 1초  | 1.0 bar 돌파 → 수만 핀 동시 개방   | 압력 정상 궤도로 뚝
1~5분    | 빠져나간 물만 보충 (안정화)        | 평탄
```

**핵심 포인트**:
- 펌프가 **아침마다 미세 소금벽을 부수고 물을 밀어내야 함** → 초기 저항(압력) 급격히 상승
- 수만 핀이 1.0 bar 도달 전까지 **꽉 참고 있다가** 동시 개방 → spike 후 급락
- **이건 정상 운영 패턴**이지 이상 신호 아님

### 3-3. 정상 상태에서 펌프의 역할
수만 핀이 열린 후 펌프는 더 이상 1.0 bar라는 '벽'과 싸우지 않음. 대신 **"빠져나간 물의 양만큼만 뒤에서 채워주는 역할"**. 필요한 압력은 거대한 힘이 아니라 배관 마찰 저항(Friction Loss)만 이겨내는 수준으로 확 낮아짐.

### 3-4. 모델링 시사점
- AE 학습 데이터에서 `is_startup_spike == 1`은 **유지** (정상 과도 응답)
- IQR 꼬리 절단을 *일부러 사용하지 않음* — 잘라내면 매일 아침 기동 overshoot까지 이상으로 학습돼 **알람 폭주**
- 운전 모드 컨텍스트(`pump_on`, `minutes_since_startup`, `is_startup_phase`)를 AE 입력에 강제 주입(VIP)해 모델이 "이건 정상 루틴"임을 학습하게 함

## 4. 딸기 재배 환경 권장값

본 프로젝트는 **㈜우성미디어서비스 — 딸기 재배환경 센서 데이터** (AI-hub) 를 기초 자료로 사용. 딸기는 저온성 작물이라 환경 제어가 까다로움.

### 4-1. 온도 (생육 단계별)

| 생육 단계 | 주간 | 야간 |
|---|---|---|
| 정식 후 활착기 | 20~23 ℃ | 8~12 ℃ |
| 영양생장기 (잎·런너) | 18~22 ℃ | 7~10 ℃ |
| 개화·착과기 | 20~25 ℃ | 5~8 ℃ |
| 수확기 | 15~20 ℃ | 3~7 ℃ |

**중요 포인트**:
- **야간 저온 유지** → 당도 상승 + 착색 향상
- **28℃ 이상 지속** → 화아 분화 억제, 기형과 증가

### 4-2. 습도

| 시간대 | 권장 RH |
|---|---|
| 낮 | 60~70% |
| 밤 | 70~85% |

- **90% 이상** → 곰팡이 (특히 잿빛곰팡이) 폭증
- **50% 이하** → 증산 과다 → 생육 저하
- 환기 + 난방 + 제습기 연동 제어 필수

### 4-3. CO₂ 농도
- 권장: **800~1200 ppm** (대기 기본 400 ppm)
- 효과: 광합성 증가 → 수량 10~30% 증가 + 당도 상승

### 4-4. VPD (증기압 결핍)
공기가 가질 수 있는 최대 수증기량(포화)과 실제 수증기량의 차이.

- 권장 범위: **0.5 ~ 1.2 kPa**
- **너무 낮음** → 과습 → 병 발생
- **너무 높음** → 수분 스트레스

### 4-5. 양액 EC (전기전도도)

| 생육 단계 | EC (dS/m) |
|---|---|
| 활착기 | 0.8~1.2 |
| 생장기 | 1.2~1.5 |
| 개화기 | 1.5~1.8 |
| 수확기 | 1.5~2.0 |

- **EC↑** → 당도 ↑ / 과 크기 ↓

### 4-6. pH
- 적정 범위: **5.5 ~ 6.5** (이상적 5.8~6.2)
- **pH > 7** → 철 결핍 (잎 황화)
- **pH < 5** → 뿌리 손상

### 4-7. 양액 주요 성분 (ppm)

| 다량 | ppm | 미량 | ppm |
|---|---|---|---|
| 질소 (N) | 100~150 | Fe | 2~3 |
| 인산 (P) | 20~40 | Mn | 0.5 |
| 칼륨 (K) | 150~250 (당도에 매우 중요) | Zn | 0.05 |
| 칼슘 (Ca) | 100~150 (과실 단단함) | B | 0.3 |
| 마그네슘 (Mg) | 30~50 | | |

### 4-8. 관수 전략
- "**자주, 소량**"이 핵심 — 하루 5~15회 (광량에 따라 조절)
- **배액률 20~30%**
- 배액 EC > 공급 EC → **염류 축적**
- 배액 EC < 공급 EC → **양분 부족**

### 4-9. 광 환경
- 광량: 200~400 μmol/m²/s
- 일조시간: 10~14시간
- 겨울 스마트팜 → LED 보광 필수

### 4-10. 핵심 5요소 요약
딸기 스마트팜 성공 = ① 야간 저온 유지 ② VPD 안정화 ③ EC + pH 정밀 관리 ④ 과습 방지 ⑤ CO₂ 보충

## 5. 관수 성능 평가 공식 (참고)

[MDPI 논문](https://www.mdpi.com/2071-1050/17/9/3981) 기준.

### 5-1. 점적기 유량 (Dripper Flow Rate)
```
q = V / t       (L/h)
  V: 수집된 물의 부피 (mL)
  t: 수집 시간 (분)
```

### 5-2. 분포 균일도 계수 (DUC, Distribution Uniformity Coefficient)
구역 내 핀들이 얼마나 균일하게 양액을 공급하는지 평가. **막힘 발생 시 급락**.

```
DUC = (X₂₅ / X_med) × 100  (%)
  X₂₅:  하위 25% 평균 유량
  X_med: 전체 평균 유량
```

| DUC | 평가 |
|---|---|
| > 90% | 우수 |
| 85~90% | 양호 |
| 65~85% | 허용 가능 |
| < 65% | **불량 — 막힘 점검 필요** |

### 5-3. 슈하르트 관리도 (Shewhart Control Chart)
DUC 등을 통계적으로 모니터링해 막힘 같은 통제 이탈 상태 감지.

```
관리 상한선/하한선 = X̄̄ ± L · (MR / d₂)
  X̄̄: 측정값 평균들의 평균 (Center line)
  L:  중심선에서 관리 한계까지의 거리 (보통 2σ 또는 3σ)
  MR: 데이터 진폭의 평균
  d₂: 상수 (n=2일 때 1.128)
```

→ 본 프로젝트의 **6시그마 3단계 알람**과 동일 통계 원리.

### 5-4. 펌프 수력 전력 / 효율
```
P_hyd = ρ · g · Q · H        (수력 전력)
  ρ: 양액 밀도, g: 중력가속도, Q: 유량, H: 양정

η = P_hyd / P_elec           (모터 펌프 효율)
  P_elec: 펌프에 공급된 전기 전력
```

→ 본 프로젝트의 `wire_to_water_efficiency` 파생변수 산출 근거.

## 6. 스마트팜 산업 3대 페인포인트 (참고)

본 프로젝트가 직접 다루는 것은 **3순위(데이터는 있지만 정답 없음)** 의 일부지만, 전체 맥락 이해를 위해 정리.

| 우선순위 | 핵심 문제 | AI 적용 가능성 |
|---|---|---|
| **1위** | 막대한 에너지 비용 + 늪에 빠진 ROI (1ha당 30~40억, 겨울 광열비 30~40%) | 에너지·환경 제어 최적화 AI |
| **2위** | 폐쇄 생태계의 역습 (병해충 단기 집단 폐사, 2019년 PLS로 농약 제한) | 비전(Vision) 탐지 AI |
| **3위** | **데이터는 넘치나 '정답'이 없는 역설** (인간 직관에 의존하는 아날로그 운영) | **센서 보정 AI, 자율 제어 AI ← 본 프로젝트** |

→ 본 프로젝트는 *센서 데이터로부터 막힘을 자동 감지·진단*해 운영자의 직관 의존도를 낮추는 방향으로 3순위 페인포인트에 접근.
